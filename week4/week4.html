  • Forms: Method to interact with a webpage (
    element)
    • document.forms : return an HTML collection of all the forms in the document.
    • how to get a form:
      • const form = document.forms[0];
      • const form = document.getElementByTagName('form')[0];
    • We can use the name attribute to identify a form: const Form = document.forms.formName; or const Form = document.forms['formName'];
    • Access to form controls:
      • const control = form.controlName;
      • const [control1, control2, ...] = form.elements
    • form.submit(): submit a file automatically
    • form.reset(): reset all the form controls back to their initial values
    • form.action(): set the action attribute of a form, it's sent to a different URL to be processed on the server.
    • form events:
      • focus: when an element is focused
      • blur: moves away from the form element
      • change: moves the focus away from the form element after changing it
      • submit: When the form is submitted.
    • Input fields: text, password, checkbox, radio, hidden, and file.
  • Object-Oriented Programming in JavaScript
    • Encapsulation: The inner workings are kept hidden inside the object, and only the essential functionalities are exposed.
    • Polymorphism: The same process can be used for different objects.
    • Inheritance: inherit all properties and methods.
    • Constructors functions:
      const myObject: function (parameter){
      this.attribute = value;
      this.method = function(){
      code
      }}
    • miObject.contructor: returns the constructor function that created it.
    • Static Methods: they can be used just for the class, not for the instances.
    • Prototype: add properties and methods to the class.
    • Monkey-patching: add methods to the prototype of JavaScript's built objects.
    • Create object from another object: const newObject: Object.create(myObject);
    • Arrow functions should be used when anonymous functions are required in callbacks (and they require less typing as well).
    • Composition over inheritance: Creating small objects that describe a single task or behavior and using them as the building blocks for more complex objects.
  • Modern JavaScript developments
    • Libraries: a piece of code that provides several methods that make it easier to achieve common tasks.
    • JQuery: support for older browsers. $ symbol alias for the global jQuery object.
    • Modular JavaScript: A module is a self-contained piece of code that provide functions and methods that can then be used in other files and by other modules.
      • export const VAR = "value";
      • import {VAR} from 'file.js';
    • Default export: single variable function or class in a Module that can be imported without having to be explicitly named.
    • MVC Frameworks:
      • Models are objects that implement the functionality for creating, reading, updating, and deleting specific pieces of information about the application.
      • Views provide a visual representation of the model.
      • Controllers link models and views together by communicating between them.
    • Templates: Templating languages allow HTML to be separated from the JavaScript program, making maintenance easier because theyâ€™re no longer tightly coupled.
    • Package Managers: A package is just a directory with one or more files of code, as well as a package.json file that contains information about the code in the package.
    • Webpack is a module bundler that takes all the JavaScript files used in your project and bundles them all together in one file.
